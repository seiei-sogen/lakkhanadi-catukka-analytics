# v0.1.0 リリース受け入れチェックリスト（全 31 TC）

## 結果記入の凡例

- ✅ **pass**: 期待通りの結果が得られた
- ❌ **fail**: 期待値に達しない（要修正 or 既知制限へ移送）
- 🟦 **n/a**: 当該観点が適用外
- ⏳ **pending**: 実機検証待ち（ユーザー手動実行）

## 検証手段の凡例

- 🤖 **judge**: LLM-as-judge（TASK-0017/0018）で自動判定
- 🦀 **rust**: Rust 単体テスト（`cargo test`）
- 👤 **手動**: 実機での手動検証（TASK-0019/0020）
- 📐 **構造**: SKILL.md / 文書の構造的妥当性チェック
