## 実装詳細

### 1. package.json の作成 🔵

**信頼性**: 🔵 *codd.yaml・note.md 技術スタックより*

最小依存:
- `typescript`（型チェック用）
- `vitest`（テスト・実行ランナー兼用、ESM 対応で軽量）
- `@types/node`（標準ライブラリ型）

スクリプト:
- `typecheck`: `tsc --noEmit`
- `test`: `vitest run`

### 2. tsconfig.json の作成 🔵

**信頼性**: 🔵 *interfaces.ts の strict 型を扱うため*

- `target`: ES2022
- `module`: ESNext / NodeNext
- `strict`: true
- `noImplicitAny`: true
- `include`: `skills/**/*.ts`, `docs/design/**/*.ts`（型定義参照のため）

### 3. テストランナー初期化 🟡

**信頼性**: 🟡 *設計ヒアリングRound A 残課題「TS スクリプト単体テスト方針」より、Vitest を妥当な選択肢として採用*

`vitest.config.ts`（または vite.config 等）を最小設定で作成。
- include: `skills/**/*.test.ts`
