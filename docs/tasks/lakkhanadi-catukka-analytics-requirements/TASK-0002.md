## 実装詳細

### 1. Cargo.toml の作成 🔵

**信頼性**: 🔵 *codd.yaml・note.md 技術スタックより*

最小依存: 標準ライブラリのみ（外部 crate なし）。

```toml
[package]
name = "lakkhanadi-catukka-normalize"
version = "0.1.0"
edition = "2021"

[lib]
name = "normalize"
path = "skills/lakkhanadi-catukka/scripts/normalize.rs"
```

スクリプト:
- 型チェック相当: `cargo check`
- テスト: `cargo test`

### 2. ツールチェイン設定 🔵

**信頼性**: 🔵 *Rust エコシステム標準より*

- 必要に応じ `rust-toolchain.toml` で `channel = "stable"` を固定
- `rustfmt.toml` は最小（プロジェクト規約があれば従う）
- Edition は 2021（または 2024）

### 3. テスト構成 🟡

**信頼性**: 🟡 *設計ヒアリングRound A 残課題「補助スクリプト単体テスト方針」より、Rust 標準の `#[cfg(test)]` モジュールを採用*

- `normalize.rs` の末尾に `#[cfg(test)] mod tests { ... }` を配置（または `tests/` ディレクトリで結合テスト）
- 実行は `cargo test`
- 既定では `cargo test` だけで完結する形にする（追加 crate なし）
