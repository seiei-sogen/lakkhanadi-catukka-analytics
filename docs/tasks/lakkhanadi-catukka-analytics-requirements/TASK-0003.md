## 実装詳細

### 1. 両プラットフォームのドキュメント調査 🟡

**信頼性**: 🟡 *prep.md「必須」項目（Codex CLI 仕様は変動するため最新確認が必須）*

- Claude Code skills: 公式ドキュメントから skill 配置パスとフロントマターキーを抽出
- Codex CLI skills: 最新の skill 仕様（YAML キー・起動規約）を確認
- 差分があるキーをリスト化

### 2. 共通フロントマターの設計 🔵

**信頼性**: 🔵 *設計ヒアリングQ1「単一 SKILL.md で両者をカバー」より*

- `name`: `lakkhanadi-catukka` （または同等のスラッグ）
- `description`: 明示呼び出しのみ起動するよう、ユーザーが「四相分析」「lakkhaṇa」等を直接指定したときに反応する文言（REQ-409）
- 両プラットフォーム独自のキーは optional として記述するか、別ファイルでオーバーライドする戦略を文書化

### 3. ADR の更新 🟡

**信頼性**: 🟡 *prep.md「確認事項」codd Wave 1 ADR 更新より*

`docs/governance/adr_skill_platform.md` に以下を追記:
- 決定: Claude Code skills と Codex CLI skills の両対応（REQ-402）
- 単一 SKILL.md による集約（NFR-301）
- 別 SKILL.md・シンボリックリンクではなく単一ファイル（設計ヒアリングQ1）
- 関連要件: REQ-401, REQ-402, REQ-409

### 4. 残された不確実性の記録 🟡

**信頼性**: 🟡 *両プラットフォーム仕様の流動性*

- 仕様変更があった場合の更新責任者・確認サイクルを README または ADR に記載
