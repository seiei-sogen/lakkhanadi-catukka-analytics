## 実装詳細

### 1. 出力テンプレートの全体構造定義 🔵

**信頼性**: 🔵 *REQ-002（出力構成順序の固定化）より*

SKILL.md の「出力テンプレート」セクションに、以下の順序を固定として記述する:

1. 対象の言い換え（一文）
2. 四相分析・要約表（4 行 × 3 列）
3. 四相分析・詳説（4 セクション）
4. 一言でまとめると

### 2. 対象の言い換え（一文）指示 🔵

**信頼性**: 🔵 *REQ-002・PRD「出力形式」より*

入力された対象を「観察可能な現象」として一文で言い換えるよう指示。以下の例文をテンプレートとして添える:

> 「（対象）とは、ここでは（観察可能な側面の言い換え）として扱います。」

### 3. 要約表（4 行 × 3 列）の Markdown ひな形 🔵

**信頼性**: 🔵 *REQ-003（3 列構造・4 行必須）より*

```markdown
| 項目 | パーリ語 | 要約 |
| --- | --- | --- |
| 特相 | lakkhaṇa | … |
| 作用 | rasa | … |
| 現れ方 | paccupaṭṭhāna | … |
| 近因 | padaṭṭhāna | … |
```

行順序（特相 → 作用 → 現れ方 → 近因）は固定。要約セルは 1 文以内で、最も中核となる特徴を簡潔に述べる旨を記述。

### 4. 詳説 4 セクションの見出し定義 🔵

**信頼性**: 🔵 *REQ-001・REQ-004・PRD「出力形式」より*

各セクションの見出しを以下で固定する（詳細な生成チェックリストは TASK-0011 で記述）:

```markdown
## 特相（lakkhaṇa）
…
## 作用（rasa）
…
## 現れ方（paccupaṭṭhāna）
…
## 近因（padaṭṭhāna）
…
```

パーリ語と現代語の併記順を「現代語（パーリ語）」で揃える（REQ-006, NFR-202）。

### 5. 一言まとめセクション 🔵

**信頼性**: 🔵 *REQ-010・NFR-002 より*

末尾に「## 一言でまとめると」セクションを必ず配置し、1〜2 文で全体を集約する旨を指示。要約表と詳説の両方を踏まえた表現を要求する。

### 6. 通常モード規模の指示（プレースホルダ）🟡

**信頼性**: 🟡 *NFR-002 より、長さモード詳細は TASK-0007 へ移譲*

通常モードの全体規模感（要約表 + 各 3〜5 文の詳説 + まとめ 1〜2 文）を一文で示し、長さモード判定詳細は TASK-0007 のセクションへ参照させる。
