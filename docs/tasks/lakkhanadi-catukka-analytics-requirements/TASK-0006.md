## 実装詳細

### 1. 入力種別 5 カテゴリの判定基準 🔵

**信頼性**: 🔵 *EDGE-001〜004, EDGE-201, EDGE-202 より*

以下のチェックリストを順に評価し、最初にマッチしたカテゴリで分岐する旨を SKILL.md に記述:

| カテゴリ | 判定条件 |
| --- | --- |
| `empty` | 入力が空文字または空白のみ |
| `gibberish` | 完全に non-Pali の意味不明文字列（例: `asdfqwer123`）で対象として扱える事象が抽出できない |
| `over-context` | 入力が極端に長くコンテキスト上限近傍に達する |
| `multi-target` | 並列に複数の対象が含まれる（例: 「怒りと悲しみと焦り」）|
| `normal` | 上記いずれにも該当せず、対象として扱える 1 件が抽出可能 |

### 2. empty カテゴリの応答テンプレート 🔵

**信頼性**: 🔵 *EDGE-001 / TC-CORE-E01 / TC-EDGE-001-01 より*

「分析対象を入力してください。例: 『怒り を四相分析して』」相当の促し文を返す。**4 セクション空虚分析を生成しないこと**を強調する。

### 3. gibberish カテゴリの応答テンプレート 🟡

**信頼性**: 🟡 *EDGE-002 / TC-CORE-E02 より*

「対象として扱える事象が抽出できませんでした。例えば『焦り』『集中』のような観察可能な現象として言い換えていただけますか」と返し、ユーザーに対象の言い換えを促す。

### 4. over-context カテゴリの応答テンプレート 🟡

**信頼性**: 🟡 *EDGE-003 / TC-EDGE-003-01 より*

「入力が長いため、要点を 1〜2 文で要約していただくか、最初の N 文字で対象抽出を行います」と明示。打ち切り方針を一貫させる。

### 5. multi-target カテゴリの応答テンプレート 🔵

**信頼性**: 🔵 *EDGE-004 / TC-EDGE-004-01・ヒアリングRound 2「1 依頼 1 対象」より*

最も中心的と思われる 1 対象を選定し、出力冒頭で「ここでは『◯◯』を中心に四相分析します。他の『△△』『□□』は本回答の対象外です」と明示する旨を記述。

### 6. 文章入力からの主題抽出ガイド 🟡

**信頼性**: 🟡 *EDGE-201 より*

入力対象が「文章」（例: 短編小説の抜粋）である場合、文章全体ではなく文章が主題化している現象（怒り、孤独、希望など）を 1 つ抽出してから分析する。抽出した主題は冒頭の言い換え一文で明示する。

### 7. normal カテゴリへの遷移 🔵

**信頼性**: 🔵 *データフロー1 より*

`normal` の場合、後続の長さモード判定（TASK-0007）→ 安全性判定（TASK-0008）→ 古典/応用判定（TASK-0009）→ 正規化（TASK-0010）→ 四相生成（TASK-0011）の順で処理を継続する旨を記述する。
