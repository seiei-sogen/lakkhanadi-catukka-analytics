## タスク概要

SKILL.md に「安全性判定 + 警告 + 観察的捉え直し」セクションを追加し、入力に医療診断・宗教的断定・他者評価・個人中傷の要求が含まれる場合に、冒頭警告と対象の観察的捉え直しを行うフローを記述する。専門家相談促進と観察的トーン保持を含む。`prompt` モジュールの拒否ポリシーと `analyzer` 前段の捉え直し責務に相当。

## 実装詳細

### 1. 安全性判定 4 カテゴリ 🔵

**信頼性**: 🔵 *REQ-106・REQ-405・PRD「注意」より*

| カテゴリ | 判定条件 | 例 |
| --- | --- | --- |
| `medical` | 医療診断・症状名による断定の要求 | 「私は鬱なのか四相分析して」 |
| `legal` | 法律的な是非・責任の断定要求 | 「この契約違反を四相分析して合法かどうか」 |
| `religious` | 教義としての真偽断定要求 | 「四念処は本当に正しい教えか教義として証明して」 |
| `others-eval` | 第三者の人格・性格・動機の断定要求 | 「あの人の性格を四相分析して」 |

### 2. 警告テンプレート文 🔵

**信頼性**: 🔵 *REQ-106 / REQ-405・PRD「注意」より*

該当カテゴリにマッチした場合、出力冒頭に以下のテンプレート相当の警告を必ず置く:

> 「ご注意: 医療・法律・宗教の権威としての断定は本 Skill の役割ではありません。以下は『観察可能な現象』として対象を捉え直したうえでの四相分析です。」

カテゴリに応じて警告文の主語（医療 / 法律 / 宗教 / 他者人格）を切り替えてよい旨を記述。

### 3. 対象の観察的捉え直しガイド 🔵

**信頼性**: 🔵 *REQ-106 / TC-SAFE-01, TC-SAFE-02 / NFR-203 より*

代表的な捉え直しの対応表を SKILL.md に記述:

| 元の対象 | 観察的捉え直し |
| --- | --- |
| 「鬱」（医療診断）| 「いま感じている重さ・動きにくさ」 |
| 他者の性格・人格 | 「自分がその人に対して感じる避けたさ・怒り」 |
| 教義の真偽 | 「教義の特相・作用・現れ方・近因を観察的に分析」 |
| 法的是非 | 「その状況で観察される現象としての行為・反応」 |

捉え直しは「対象の言い換え（一文）」セクション（TASK-0005）で具体化する旨を明記。

### 4. 専門家相談促進テンプレート 🔵

**信頼性**: 🔵 *REQ-201・REQ-405 / TC-SAFE-01 より*

医療・法律・宗教カテゴリでは、出力末尾近く（一言まとめの直前または直後）に以下相当の一文を必ず添える:

> 「医療的な判断・法的助言・教義の正統性については、専門家・原典・信頼できる先生に相談・照合してください。」

### 5. 観察的トーン保持の指示 🔵

**信頼性**: 🔵 *NFR-203 / TC-NFR-201-01・REQ-407（observation-first principle）より*

以下を SKILL.md に明記:

- 命令形（「〜すべき」「〜してはいけない」）を避ける
- 断定的非難語（「悪い」「ダメだ」「異常だ」）を避ける
- 「〜と観察できる」「〜として現れる」のような観察動詞を選ぶ
- 即時の善悪判断を行わず、まず現象として記述する（REQ-407）

### 6. 安全性判定後の処理フロー 🔵

**信頼性**: 🔵 *REQ-106（拒否せず警告 + 捉え直しで継続）より*

該当カテゴリにマッチしても **拒否はせず** 、警告 + 捉え直し + 専門家相談促進を加えたうえで、通常の四相分析フローへ継続する旨を記述。
