## 実装詳細

### 1. 古典明示トピックの代表例リスト 🔵

**信頼性**: 🔵 *REQ-104・lakkhanadi-catukka-example.ncl の examples（地大・水大・火大・眼浄色・色境）より*

以下を「古典明示」として SKILL.md に列挙する（網羅ではなく代表例）:

- 四大: pathavī-dhātu（地）/ āpo-dhātu（水）/ tejo-dhātu（火）/ vāyo-dhātu（風）
- 五感の透明色: cakkhu-pasāda（眼）/ sota-pasāda（耳）等
- 心所（cetasika）: phassa（触）/ vedanā（受）/ saññā（想）等
- その他、原典・注釈書に直接記述のある仏教概念

判定の最終決定は LLM が「古典に直接記述があるか」を観点に行う旨を明記。

### 2. 応用マーカーの文言と配置 🔵

**信頼性**: 🔵 *REQ-105・PRD「重要な方針」より*

応用と判定した場合、出力冒頭（言い換え一文の直前または同じ段落内）に以下を必ず付与:

> 「四相分析の枠組みを応用すると、…」

応用マーカー付き出力では、表現が「古典の断定」ではなく「枠組みの応用」であることを LLM 自身が認識して記述する旨を強調。

### 3. 両義的トピックの併記ガイド 🟡

**信頼性**: 🟡 *EDGE-102 / TC-MARK-B01 より*

仏教用語と現代心理用語の中間に位置するトピック（例: 「サマーディ（集中）」「タンハー（渇望 / 執着）」）については、応用マーカーを付けたうえで:

> 「古典では〜とされ、現代的には〜と扱える」

の併記表現を採用する旨を記述。

### 4. 技術概念の観察対象化指示 🔵

**信頼性**: 🔵 *EDGE-202 / TC-EDGE-202-01・PRD「技術概念にも応用してよい」より*

入力対象が技術概念（コンパイラ、TCP、キャッシュミス、燃え尽き症候群等）の場合、応用マーカーを付けたうえで以下のように扱う:

- 技術現象を「観察対象（現象）」として捉える
- 仕様書ではなく「現れ方」「条件」を記述する
- 古典的解釈と混同しない（応用マーカーが付与されることで明示）

### 5. 判定フローと出力への反映 🔵

**信頼性**: 🔵 *REQ-104, REQ-105・データフローより*

判定結果は出力テンプレート（TASK-0005）の冒頭部分に反映される。古典の場合はマーカーなし、応用の場合はマーカー付きで言い換え一文を生成する旨を明記。

### 6. 古典判定の保守性原則 🟡

**信頼性**: 🟡 *REQ-406（教義として確認できない内容を断定しない）から妥当な推測*

判定に迷う場合は **応用扱い** にフォールバックする（誤って古典と断定するリスクを避ける）。ヒアリングRound 2「閾値ベース自動」と整合。
