### 4. TC-NORM-03: ダイアクリティカルなし表記の正規化 🔵

**信頼性**: 🔵 *acceptance-criteria.md TC-NORM-03 より直接*

```rust
#[test]
fn normalizes_iast_diacritic_drift() {
    assert_eq!(
        normalize("lakkhana, paccupatthana の意味"),
        "lakkhaṇa, paccupaṭṭhāna の意味"
    );
}
```

### 5. TC-NORM-B01: 境界値（崩れた異形） 🟡

**信頼性**: 🟡 *acceptance-criteria.md TC-NORM-B01「推測まで採用していないため標準表記に近いものは正規化、極端な崩れは不明な表記として扱う」より*

```rust
#[test]
fn lets_extreme_drift_pass_through() {
    // 極端な崩れ "padaaaaana" は変換せずそのまま通す
    assert_eq!(normalize("padaaaaana"), "padaaaaana");
}

#[test]
fn near_standard_with_uppercase_is_handled_per_rule_table() {
    // 大文字 T を含むが、ルール `padatthana` (小文字) と case-insensitive で一致する想定
    // 実装の方針に依存するため、ルール表に padatthana が含まれていれば正規化、なければスルー
    let result = normalize("padaTthana");
    assert!(result == "padaṭṭhāna" || result == "padaTthana");
}
```

> 備考: TC-NORM-B01 は「推測まで採用していない」ため、ルール表に明示登録された異形は case-insensitive で正規化、それ以外（`padaaaaana` 等）は通す、という二極の挙動を確認する。

### 6. 大文字小文字混在ケース 🟡

**信頼性**: 🟡 *interfaces.ts コメント「大文字小文字を区別しないマッチを行うことを推奨」より妥当な推測*

#[test]
fn normalizes_capitalized_variant_to_lowercase_standard() {
