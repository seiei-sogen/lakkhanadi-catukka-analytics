```ts
  describe('境界値', () => { /* TC-NORM-B01 */ });
  describe('複合・文中混在', () => { /* 補助テスト */ });

```

### 2. TC-NORM-01: 既知異形 padahāna → padaṭṭhāna 🔵

**信頼性**: 🔵 *acceptance-criteria.md TC-NORM-01 より直接*

```ts
it('padahāna を padaṭṭhāna に正規化する', () => {
  expect(normalize('padahāna について教えて')).toBe('padaṭṭhāna について教えて');
});
```

### 3. TC-NORM-02: 既知異形 paccupahāna → paccupaṭṭhāna 🔵

**信頼性**: 🔵 *acceptance-criteria.md TC-NORM-02 より直接*

```ts
it('paccupahāna を paccupaṭṭhāna に正規化する', () => {
  expect(normalize('paccupahāna について')).toBe('paccupaṭṭhāna について');
});
```

### 4. TC-NORM-03: ダイアクリティカルなし表記の正規化 🔵

**信頼性**: 🔵 *acceptance-criteria.md TC-NORM-03 より直接*

```ts
it('lakkhana, paccupatthana を標準表記に正規化する', () => {
  expect(normalize('lakkhana, paccupatthana の意味'))
    .toBe('lakkhaṇa, paccupaṭṭhāna の意味');
});
```

### 5. TC-NORM-B01: 境界値（崩れた異形） 🟡

**信頼性**: 🟡 *acceptance-criteria.md TC-NORM-B01「推測まで採用していないため標準表記に近いものは正規化、極端な崩れは不明な表記として扱う」より*

```ts
it('極端な崩れ "padaaaaana" は変換せずそのまま通す', () => {
  expect(normalize('padaaaaana')).toBe('padaaaaana');
});

it('標準表記に近い "padaTthana" は case-insensitive で正規化される', () => {
  // 大文字 T を含むが、ルール `padatthana` (small letter) と case-insensitive で一致する想定
  // 実装の方針に依存するため、ルール表に padatthana が含まれていれば正規化、なければスルー
  const result = normalize('padaTthana');
  expect([result === 'padaṭṭhāna', result === 'padaTthana']).toContain(true);
});
```

> 備考: TC-NORM-B01 は「推測まで採用していない」ため、ルール表に明示登録された異形は case-insensitive で正規化、それ以外（`padaaaaana` 等）は通す、という二極の挙動を確認する。

### 6. 大文字小文字混在ケース 🟡

**信頼性**: 🟡 *interfaces.ts コメント「大文字小文字を区別しないマッチを行うことを推奨」より妥当な推測*

```ts
it('文頭大文字始まりの異形も正規化される', () => {
  expect(normalize('Padahāna は重要')).toBe('padaṭṭhāna は重要');
  // 注: 出力の大文字保持は MVP 範囲外。小文字標準表記に統一する方針
});
```

### 7. 複数同時出現・文中混在ケース 🔵

**信頼性**: 🔵 *TC-NORM-03 が複数語のケースを示しているため*

```ts
it('1 文中に複数の異形が混在しても全て正規化される', () => {
  expect(normalize('lakkhana と paccupatthana を比較')).toBe('lakkhaṇa と paccupaṭṭhāna を比較');
});

it('既知異形と IAST 揺れが同居しても両方正規化される', () => {
  expect(normalize('padahāna と lakkhana')).toBe('padaṭṭhāna と lakkhaṇa');
});
```

### 8. ルールに該当しない入力のスルー 🔵

**信頼性**: 🔵 *interfaces.ts コメント「該当しない極端な崩れはそのまま通し下流の SKILL.md (LLM) 側で unknown_term として扱わせる」より*
