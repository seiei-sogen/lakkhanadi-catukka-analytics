### 2. Skill 推論実行手順 🔵

**信頼性**: 🔵 *REQ-408「--tools "" 起動」・dataflow.md フロー 7 より*

各サンプルについて Claude Code（または同等の AI コマンド）で SKILL.md を実行し、出力を保存する。実行コマンド例:

```bash
# 例: Codex / Claude Code 共通の使い方（実装フェーズで確定）
claude --print --model claude-opus-4-6 --tools "" \
  --skill skills/lakkhanadi-catukka \
  < skills/lakkhanadi-catukka/tests/samples/core-classical.md \
  > /tmp/output-core-classical.md
```

出力 7 ファイルを `/tmp/judge-run-<timestamp>/output-*.md` 配下に保存する（git 管理対象外）。

### 3. judge 実行手順 🔵

**信頼性**: 🔵 *TASK-0017 SKILL_JUDGE.md・interfaces.ts JudgeReport より*

各 Skill 出力に対して SKILL_JUDGE.md を実行し、JSON レポートを取得:

```bash
# 入力ペイロード作成（Skill 出力 + 期待値）
cat > /tmp/judge-input-core-classical.md <<EOF
## 評価対象
### Skill 出力
$(cat /tmp/output-core-classical.md)
### 期待モード
normal
### 期待 canonicality
canonical
### 期待 safety
normal
EOF

# judge 実行
claude --print --model claude-opus-4-6 --tools "" \
  < skills/lakkhanadi-catukka/tests/judge/SKILL_JUDGE.md \
  --append /tmp/judge-input-core-classical.md \
  > /tmp/judge-result-core-classical.json
```

実コマンド形式は実装時に確定するが、`--tools ""` 制約（REQ-408）下で動く形式とする。

### 5. 違反箇所の SKILL.md フィードバックループ手順 🔵

**信頼性**: 🔵 *NFR-302「品質チェックリストに対する判定」より*

`overallPass: false` のサンプルがあった場合:
1. judge レポートの `aspects[].reason` から違反観点を特定
2. 対応する SKILL.md セクション（TASK-0005〜0012 のいずれか）に修正を加える
   - `concrete-examples` fail → TASK-0011 の生成チェックリスト
   - `detail-sentence-count` fail → TASK-0007 の長さモード指示
   - `applied-marker` fail → TASK-0010 の応用マーカー判定指示
   - `safety-advisory` fail → TASK-0009 の警告フロー
   - `summary-table-rows` / `four-sections-coverage` fail → TASK-0005 のテンプレート
3. 該当サンプルのみ再実行 → 再判定 → pass を確認
4. リグレッション防止のため、修正後に他 6 サンプルも再実行（影響範囲確認）

### 6. judge 自体のばらつき確認 🔵

**信頼性**: 🔵 *NFR-302「自動評価」の信頼性確保より*

- 3 回とも全観点 pass であれば再現性あり
- ばらつきがある観点を特定し、SKILL_JUDGE.md の判定指示を強化する（TASK-0017 へフィードバック）

## 注意事項

- LLM 推論にはコストと時間が掛かるため、サンプル実行と judge 実行はまとめてバッチ的に行う（dataflow.md「バッチ処理」備考）
- judge レポートは git 管理対象外とし、`/tmp/judge-run-<timestamp>/` などに格納
- SKILL.md への修正は最小限とし、修正のたびに該当サンプル + リグレッション 6 サンプルを再判定する
- 「許容違反」とする場合は明示的な根拠（要件未定義の境界等）を集計レポートに記録する
- judge 自体の判定揺れは TASK-0017 の SKILL_JUDGE.md に逆フィードバック（観点指示の明確化）
