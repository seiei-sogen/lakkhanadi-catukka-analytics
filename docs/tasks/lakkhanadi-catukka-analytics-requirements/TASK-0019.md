## 実装詳細

### 1. Claude Code skills 配置 🔵

**信頼性**: 🔵 *NFR-301・TASK-0003 で確定したフロントマター仕様より*

- TASK-0003 で調査・確定した Claude Code skills の配置パスとフロントマター仕様に従う
- `skills/lakkhanadi-catukka/SKILL.md` がそのまま読み込まれる構成
- 必要に応じてシンボリックリンクや plugin マニフェストを追加

### 2. 明示呼び出しテスト 🔵

**信頼性**: 🔵 *REQ-409・TC-PLAT-04・design-interview ヒアリングRound 2「起動トリガー」より*

トリガーとなる呼び出し例：
- 「lakkhanadi-catukka で pathavī-dhātu を分析して」（skill 名指名）
- 「四相分析を実行して: 嫉妬」
- 「lakkhaṇa rasa paccupaṭṭhāna padaṭṭhāna で焦りを分析」

非起動の確認：
- 「今日の天気は？」
- 「Rust で関数の引数チェックを書いて」
- description が「四相分析」「lakkhaṇa」等を含む明示要求でのみ反応する文言になっていることを確認

### 3. 日本語固定出力検証 🔵

**信頼性**: 🔵 *REQ-404・TC-PLAT-03 より*

- 英語入力サンプルで起動: "Please do a four-phase analysis (lakkhanadi-catukka) of TCP connection."
- 出力本文がすべて日本語であることを確認
- 要約表ヘッダ（項目 / パーリ語 / 要約）も日本語

### 4. 7 サンプルでの動作確認 🔵

**信頼性**: 🔵 *TASK-0016 のサンプル設計・acceptance-criteria.md より*

`tests/samples/` の 7 ファイル（core-classical / core-psychological / core-technical / len-short / len-detailed / safety-medical / norm-iast）を順に Claude Code 経由で実行し、出力をログに保存。

### 5. 前処理フック動作確認 🟡

**信頼性**: 🟡 *architecture.md「補助構成」備考・REQ-408 との両立確認より*

- Claude Code skills が `scripts/normalize.rs`（ビルド済みバイナリ）を前処理として呼び出す機構を持つ場合は、norm-iast.md 入力で正規化が走ることを確認
- 持たない場合は SKILL.md 内テーブル（TASK-0010）でフォールバック動作することを確認
- どちらの経路でも TC-NORM-03 相当の出力が得られる

### 6. 検証ログの保存 🟡

**信頼性**: 🟡 *運用上のドキュメント方針として妥当な追加*

`docs/tasks/lakkhanadi-catukka-analytics-requirements/verification/claude-code/` を作成し、各サンプルに対する入力・出力ペアと簡易判定結果（手動 + judge）を保存する。
