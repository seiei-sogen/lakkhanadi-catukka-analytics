同じ `skills/lakkhanadi-catukka/` ディレクトリを Codex CLI skills として認識・呼び出し、Claude Code（TASK-0019）と同等の出力が得られるかを確認する。前処理フックの有無による `scripts/normalize.ts` の挙動差異も確認し、二段冗長（TS スクリプト経由 / SKILL.md 内テーブルでのフォールバック）が機能していることを実機で検証する。REQ-408 の `--tools ""` 制約下での動作も含む。

## 実装詳細

### 1. Codex CLI skills 配置 🔵

**信頼性**: 🔵 *TASK-0003 の調査結果・REQ-402・NFR-301 より*

- TASK-0003 で確定した Codex CLI skills の配置仕様に従う
- 単一 SKILL.md で両プラットフォームをカバー（架空の差分ファイルを置かない）
- 必要に応じて Codex CLI 固有マニフェスト（plugin.toml 等）の最小ファイルを追加

### 2. 7 サンプルでの動作確認 🔵

**信頼性**: 🔵 *TASK-0016 のサンプル設計・REQ-402 より*

7 サンプルを Codex CLI 経由で実行し、出力ログを保存。Claude Code 出力（TASK-0019）と並べて比較し、以下の観点で同等性を評価：
- 4 セクション網羅
- 詳説文数（モード別）
- 応用マーカー有無
- 警告 + 観察的捉え直し（safety-medical サンプル）
- 正規化結果（norm-iast サンプル）

### 3. 前処理フック対応環境での normalize.ts 動作確認 🟡

**信頼性**: 🟡 *architecture.md「補助構成」備考・実装フェーズで挙動確認すべき項目*

- Codex CLI が前処理フックを提供する場合は、TS スクリプト経由で正規化が走ることを確認
- TC-NORM-01 〜 TC-NORM-03 相当の入力で、SKILL.md に渡る前に正規化済みになっているかを検証

### 4. 前処理フック非対応環境でのフォールバック確認 🔵

**信頼性**: 🔵 *architecture.md「補助構成」備考・REQ-408 との両立より*

- 前処理フックを持たない環境（または手動で TS をスキップ）で、SKILL.md 内テーブル（TASK-0010）が単独で正規化を実行できることを確認
- 入力「padahāna について教えて」が出力で「padaṭṭhāna」として扱われる

### 5. `--tools ""` 制約下での動作 🔵

**信頼性**: 🔵 *REQ-408・codd.yaml `ai_command` より*

- `claude --print --model claude-opus-4-6 --tools ""` 相当の実行モードで動作することを確認
- skill 自身がツールを呼ばない構成（TASK-0004 の方針）が機能している
- LLM が外部 API・ファイル読み込みを試みていないこと

### 6. 検証ログの保存と差分レポート 🟡

**信頼性**: 🟡 *運用上のドキュメント方針として妥当な追加*

- Claude Code 出力（TASK-0019）と Codex CLI 出力を並べた差分レポートを作成
- 同じサンプルで顕著な差（4 相のうち 1 つが欠落、応用マーカーの有無逆転、トーンが大きく異なる）が出ていれば、SKILL.md / scripts/normalize.ts にフィードバック
- 軽微な文言差（語彙選択・文長）は許容
